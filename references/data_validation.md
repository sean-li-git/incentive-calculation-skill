# 数据校验规则参考

> Phase 2 执行数据校验时按需查阅。

---

## 通用校验清单

### 1. 结构检查

```
- [ ] Excel 文件可正常打开，无损坏
- [ ] 各 sheet 名称和用途已识别
- [ ] 表头行已定位（注意可能有合并单元格或多行表头）
- [ ] 数据区域已确定（排除空行、汇总行、备注行）
```

### 2. 人员维度

```
- [ ] 人员唯一标识字段已确定（工号 / 姓名 / 身份证号）
- [ ] 无重复人员记录（或已明确重复原因和处理方式）
- [ ] 人员信息表与业绩数据表的人员范围一致
  - 有业绩无人员信息的 → 无法计算，标记 🔴
  - 有人员信息无业绩的 → 确认是否应参与（可能金额为0）
```

### 3. 数值字段

```
- [ ] 数值字段中无文本值（如 "N/A"、"-"、"待确认"）
- [ ] 无意外的负数
- [ ] 无明显异常值（可用 IQR 或 Z-score 快速筛查）
- [ ] 百分比字段的表达方式统一（0.85 vs 85% vs 85）
- [ ] 金额单位统一（元 vs 万元）
```

### 4. 业务逻辑

```
- [ ] 达成率/完成率的计算基准明确（目标值来源确认）
- [ ] 时间维度正确（月度/季度/年度数据与核算周期匹配）
- [ ] 分组维度与方案一致（部门/团队/BU的划分口径）
```

---

## 常见数据问题及处理方式

| 问题 | 典型表现 | 建议处理 |
|------|---------|---------|
| 合并单元格 | 部门名只在第一行，下面为空 | 向下填充 |
| 多行表头 | 前2-3行都是表头信息 | 合并为单行表头或跳过 |
| 混合数据类型 | 数字列中有"-"或"/"表示无数据 | 统一替换为空值或0（需确认） |
| 百分比不统一 | 同列中有 0.85 和 85% | 统一为小数或百分比 |
| 隐藏行/列 | 数据看起来不完整 | 检查是否有隐藏内容 |
| 公式残留 | 单元格显示值与实际值不同 | 以实际值为准 |
| 汇总行混在数据中 | "合计"行在数据区域内 | 识别并排除 |

## 计算结果校验方法

1. **总额校验**：各人金额求和 = 总奖金包（如方案有总额限制）
2. **抽样手算**：随机抽 3-5 人手工复核完整计算过程
3. **极值检查**：最高/最低金额是否合理
4. **分布检查**：金额分布是否符合预期（如是否过于集中或分散）
5. **零值检查**：金额为 0 的人员是否应该为 0（而非计算错误）
