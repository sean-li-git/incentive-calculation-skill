# 激励核算执行指南

> **定位**：方案已确定，进入计算执行阶段。本 skill 不做方案设计，只做核算。
> 输出的 Excel 文件遵循咨询级格式标准，所有计算保留公式可追溯。

---

## 核心工作流（三阶段）

```
Phase 1 理解方案 → Phase 2 数据校验 → Phase 3 执行计算
    ↓ 用户确认        ↓ 用户确认          ↓ 输出结果
```

**关键纪律：不跳阶段。** 方案没理解清就不碰数据，数据没校验完就不开始算。

---

### Phase 1：理解激励方案

**输入**：用户以任意形式提供激励方案（自然语言描述 / Word-PDF文档 / Excel模板 / 口头说明）。

**执行步骤**：

1. 阅读并解析方案全文
2. 提取并结构化以下要素（缺失项主动追问）：

| 要素 | 说明 |
|------|------|
| 激励类型 | 业绩奖金 / 项目奖金 / 其他 |
| 计算公式 | 完整的计算逻辑链，含每一步的输入、运算、输出 |
| 所需数据字段 | 从公式反推：计算需要哪些原始数据（如基本工资、销售额、回款率等） |
| 特殊规则 | 封顶/保底、阶梯系数、分段计算、四舍五入规则等 |
| 适用范围 | 哪些人参与、排除条件（如有） |
| 时间维度 | 核算周期、数据时间范围 |

3. 用清晰的结构化格式向用户复述理解结果：
   - 用公式或伪代码表达计算逻辑
   - 列出完整的**数据需求清单**（字段名 + 用途 + 数据类型）
   - 标注所有不确定或可能有歧义的地方，明确提出

**退出条件**：用户确认理解无误。

---

### Phase 2：数据接收与校验

**输入**：用户提供 Excel 文件（可能是一张或多张表）。

**执行步骤**：

1. **读取数据**：使用 openpyxl 读取 Excel 文件，识别各 sheet 的内容和结构

2. **字段映射**：将 Excel 中的实际列名与 Phase 1 中确定的数据需求清单逐一对照
   - 能直接对应的：标记为 ✅
   - 名称不同但含义可能一致的：标记为 ⚠️ 并向用户确认
   - 缺失的字段：标记为 ❌ 并提醒用户补充

3. **数据质量检查**（逐项执行并报告，详见 [references/data_validation.md](references/data_validation.md)）：
   - 完整性检查：必要字段空值、人员覆盖范围
   - 唯一性检查：重复记录
   - 合理性检查：数值范围、数据类型
   - 一致性检查：数据口径、时间范围

4. **输出数据校验报告**：
   - 🔴 阻断问题（必须解决才能继续）
   - 🟡 需确认问题（可能影响结果，需用户判断）
   - 🟢 提示信息（不影响计算，但值得注意）

5. **整理计算用数据集**

**退出条件**：所有 🔴 问题已解决，🟡 问题已由用户确认处理方式。

---

### Phase 3：执行计算与输出

**输入**：Phase 1 确认的方案逻辑 + Phase 2 整理好的数据集。

**执行步骤**：

1. **样例验证**：先展示 2-3 个样例人员的逐步计算过程，让用户验证逻辑正确性

2. **批量计算并生成 Excel 文件**（使用 openpyxl，遵循下方格式标准）：

   **⚠️ 核心要求：所有计算必须保留 Excel 公式，不写死数值。**

   **Sheet 结构**（详见 [references/incentive-templates.md](references/incentive-templates.md)）：

   | Sheet | 内容 |
   |-------|------|
   | **原始数据** | 从用户 Excel 提取的业绩数据和人员信息，作为公式数据源 |
   | **个人明细表** | 每人一行，公式引用原始数据 → 中间计算步骤 → 最终金额 |
   | **部门汇总表** | 按分组维度汇总，使用 SUMIF/COUNTIF/AVERAGE 等公式 |
   | **校验报告** | 异常值标注、总额校验、抽样复核（如需要） |

3. **结果概览**：在对话中汇报关键数字（参与人数、总金额、人均金额、分组分布）

## Excel 格式标准

所有输出 Excel 文件必须遵循以下格式标准。详细代码模式见 [references/openpyxl-patterns.md](references/openpyxl-patterns.md)。

### 颜色编码

| 字体颜色 | RGB | 含义 | 示例 |
|----------|-----|------|------|
| **蓝色** | #0000FF | 硬编码输入值 / 原始数据 | 基本工资: 15000 |
| **黑色** | #000000 | 同 Sheet 内公式 | `=B2*C2` |
| **绿色** | #008000 | 跨 Sheet 引用 | `='原始数据'!B2` |
| **红色** | #FF0000 | 异常标记 / 待确认 | 金额为负数 |

### 填充色

| 填充 | Hex | 用途 |
|------|-----|------|
| 深藏青 | #1F4E79 | 区域标题行（白色加粗文字） |
| 浅蓝 | #D9E1F2 | 列标题 / 子标题 |
| 浅黄 | #FFF2CC | 关键输入单元格高亮 |
| 中蓝 | #BDD7EE | 输出 / 汇总行 |

### 排版规范

- **字体**: Calibri 10pt（正文），11pt（标题）
- **加粗**: 汇总行（合计、均值等）和区域标题
- **边框**: 小计行上方单线，总计行下方双线
- **冻结窗格**: 固定表头行，方便滚动查看
- **打印设置**: 横向，适合页宽

### 公式纪律

1. **不嵌入魔法数字**: `=B2*0.15` 是错的 → 用 `=B2*$G$1` 引用系数单元格
2. **中间步骤可见**: 复杂计算拆分为多列，每列一个公式步骤
3. **汇总用函数**: 合计用 `SUM`，条件汇总用 `SUMIF`/`COUNTIF`，不手写数值

## 行为准则

1. **不假设，多确认**：方案逻辑有任何不确定的地方，停下来问，不要自行猜测
2. **可追溯**：输出 Excel 中所有计算结果必须是公式而非硬编码数值
3. **先样例后批量**：批量计算前先用样例验证，避免全量错误
4. **数据原样保留**：不修改原始数据文件，所有整理和计算在新文件中进行
5. **主动报告异常**：计算过程中发现的任何异常立即告知用户

## 数据保密处理

> 薪酬激励数据高度敏感，核算过程中必须重视数据保密。

**在 Phase 2 开始前，主动询问用户数据保密需求，并根据情况采用以下策略：**

### 策略 1：数据脱敏（推荐）
- 姓名替换为"员工A/B/C"或随机编号
- 删除或替换工号、身份证号等敏感标识字段
- 金额可按统一比例缩放（不影响计算逻辑验证）

### 策略 2：模拟数据
- 用户只提供方案逻辑和数据结构
- 按结构生成模拟数据，先跑通整个计算流程
- 用户拿验证通过的计算逻辑和模板，自行套用真实数据

### 策略 3：混合模式
- Phase 1 和 Phase 3 用模拟数据
- Phase 2 用户只提供表头和少量样例行
- 最终批量计算由用户在本地用真实数据执行

**原则：不主动索要敏感数据，用户未脱敏的数据不存储、不外传。**
